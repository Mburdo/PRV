The Matching Algorithm

PRV uses a "step-ladder" approach: fast, simple checks first, falling through to more sophisticated analysis only when needed. This achieves 100% accuracy while maintaining sub-500ms performance.

Time Window Filter

Before any matching begins, filter candidate sessions by time proximity to the commit.

WHERE session.started_at <= commit.time
AND commit.time <= session.ended_at + 30min
AND session.workspace = commit.repo_path

File Path Overlap
